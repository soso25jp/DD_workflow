Hướng dẫn thực hiện tài liệu DD

Tài liệu được sắp xếp lại theo đúng trình tự công việc, giúp xác định phạm vi đối ứng, kiểm tra tài liệu cần sửa, thực hiện cập nhật và theo dõi tiến độ một cách rõ ràng, chuyên nghiệp.

Mục lục

1. Xác định phạm vi đối ứng 2. Bắt đầu công việc & cập nhật tiến độ 3. Cách tìm tài liệu liên quan 4. Cách viết tài liệu .doc 5. Cách viết tài liệu .excel 6. Thứ tự làm việc đề xuất

1. Xác định phạm vi đối ứng

Bước 1: Kiểm tra file 機能変更対応マトリクス（担当会社プロットVer）

  • Tìm các đối tượng IK / IR / command mà mình phụ trách thực hiện.
  • Các nội dung được đánh dấu tròn (〇) là các hạng mục cần đối ứng.

Bước 2: Kiểm tra file DD_機能-成果物紐づけ一覧

Tại sheet 機能_対象資料一覧, thực hiện các bước sau:

  1. Kiểm tra cột [担当] để xác nhận các tài liệu mình phụ trách.
  2. Kiểm tra cột [修正対象] để xác định mức độ cần sửa của từng tài liệu.
  3. Sau khi kiểm tra, ghi kết quả vào cột [確認結果].
Ký hiệu Ý nghĩa ở cột [修正対象] Giá trị ghi vào [確認結果]
〇 Bắt buộc phải sửa 修正対象
△ Cần mở tài liệu ra kiểm tra, nếu cần thì sửa 必要あれば修正
× Không sửa 修正なし

2. Bắt đầu công việc và cập nhật tiến độ

Bước 3: Cập nhật tiến độ trong quá trình làm

Khi bắt đầu công việc, cập nhật phần trăm tiến độ vào các cột sau:

  • [作成/修正]
  • [自己レビュー]
  • [FPT内部レビュー]
  • [NECレビュー]

Quy tắc cập nhật phần trăm

Hạng mục Giá trị Ý nghĩa
[作成/修正], [自己レビュー] 0% ～ 100% Ghi theo mức độ hoàn thành thực tế, 100% là hoàn tất
[FPT内部レビュー], [NECレビュー] 0% Chưa ở trạng thái có thể review
[FPT内部レビュー], [NECレビュー] 80% Đang chờ review
[FPT内部レビュー], [NECレビュー] 90% Đang sửa lỗi theo comment review
[FPT内部レビュー], [NECレビュー] 100% Review hoàn tất
Lưu ý: Sau khi sửa xong các chỉ摘, cần re-request review.

3. Cách tìm và đọc tài liệu liên quan

  • Tham khảo file: Sơ đồ mapping tài liệu onboarding.
  • Dùng file này để tra cách đọc và tìm tài liệu tương ứng khi thực hiện DD.

4. Cách viết tài liệu .doc

4.1. Nguyên tắc cơ bản

  • Với TBL cũ, cần update thêm TBL mới tương ứng ở bên cạnh để đối chiếu.
  • Để xác định TBL mới tương ứng, xem chi tiết tại sheet 項目一覧 trong file L2900_0400DB仕様IDアカウント管理(IK).

4.2. Cách mapping bảng

Ví dụ:

  • Tìm 項目名 của tài liệu cần làm.
  • Mapping từ 2 bảng cũ:
    • 新docomoID発行者情報
    • 新docomoID未発行者情報
  • Sang 3 bảng mới:
    • dアカウント情報
    • 携帯事業
    • CF事業

Các ô có dấu tròn (〇) trong bảng là các quan hệ mapping tương ứng.

4.3. Cách ghi note trong tài liệu

Ví dụ mapping cụ thể:

DB表「新docomoID発行者情報」 → DB表「dアカウント情報」

Ghi note:

DB表「新docomoID発行者情報」、DB表「新docomoID未発行者情報」は27v1long移行完了後から使用しない。

Trường hợp bảng không còn sử dụng, ghi note:

27v1long移行後は、使用しないプロセス

4.4. Tài liệu tham khảo

  • Tham khảo IK509 để xem cách viết chi tiết.

5. Cách viết tài liệu .excel

5.1. Bối cảnh và vấn đề chính

Hệ thống hiện tồn tại 2 luồng xử lý:

  • 旧ルート (Old route): xử lý theo DB / Table cũ
  • 新ルート (New route): xử lý theo DB / Table mới

Khi request vào hệ thống (văn bản / 電文), cần quyết định dùng xử lý cũ hay xử lý mới.

5.2. Thiết kế / hướng xử lý

A

Tạo 共通ルート

Gom những phần xử lý mà dù đi old hay new route đều cần dùng, sau đó mới phân nhánh.

  • 電文パラメータチェック
  • DB異常時のリトライループ処理
B

Common processing do vendor khác thực hiện

Vendor khác cung cấp search + decision để xác định đi old route hay new route.

C

Chính sách phân nhánh

Có nhánh từ 共通ルート sang 旧ルート hoặc 新ルート. Không làm nhánh 旧→新 trong scope lần này.

5.3. Phân loại trạng thái user

  1. 未移行: chưa migrate, dữ liệu còn ở old → đi 旧ルート
  2. 移行済み: đã migrate → đi 新ルート
  3. 新規ユーザー: user đăng ký mới → đi 新ルート

Lý do phải giữ cả 2 route là vì việc migrate không diễn ra đồng loạt và cần hạn chế chỉnh sửa vào old route càng nhiều càng tốt.

5.4. Yêu cầu về deliverable / cấu trúc sheet

Sheet Nội dung
共通ルートシート Mô tả common route, common processing, search + decision và phần phân nhánh
旧ルートシート Mô tả xử lý cho user chưa migrate, có truy cập old table
新ルートシート Mô tả xử lý cho user đã migrate / user mới, có truy cập new table

5.5. Trường hợp common processing chưa rõ chi tiết

  • Chưa xác định được search query cụ thể, điều kiện WHERE, key search, rule ưu tiên và logic decision chi tiết.
  • Vì vậy, trong tài liệu cần dùng オートシェイプ để đánh dấu phần sẽ được vendor khác bổ sung.
  • Đặc biệt với phần search processing, cần đặt placeholder thể hiện rõ sẽ có điều kiện search cho old/new.

5.6. Yêu cầu chỉnh sửa pattern matrix / plot chức năng

Dựa trên プロット表 / パターンマトリクス và nội dung tài liệu:

  • Nếu Old route / New route có phần cần đối ứng theo chức năng đang xử lý, thì bổ sung hoặc chỉnh sửa mô tả vào sheet tương ứng.
  • Ví dụ: IR006, IK...

6. Thứ tự làm việc đề xuất

  1. Kiểm tra 機能変更対応マトリクス（担当会社プロットVer） để xác định IK / IR / command cần làm.
  2. Mở DD_機能-成果物紐づけ一覧 và xác nhận tài liệu phụ trách ở cột [担当].
  3. Kiểm tra cột [修正対象] để xác định tài liệu nào bắt buộc sửa, tài liệu nào cần kiểm tra thêm.
  4. Ghi kết quả vào cột [確認結果].
  5. Tham khảo Sơ đồ mapping tài liệu onboarding để xác định cách đọc và tài liệu liên quan.
  6. Tiến hành chỉnh sửa tài liệu:
    • Nếu là .doc → mapping old/new table và bổ sung note đúng quy tắc.
    • Nếu là .excel → tổ chức nội dung theo 3 sheet: 共通 / 旧 / 新 route.
  7. Trong quá trình làm, cập nhật % tiến độ vào các cột: 作成/修正, 自己レビュー, FPT内部レビュー, NECレビュー.
